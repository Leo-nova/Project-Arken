# Baseline vs Persona (V5)- Gemini-2.5-Pro 測試報告 
**Case Study: VOO/VYM Investment Simulation Dialogue**

## 📌 Introduction
本專案比較 **Baseline** 與 **Persona** 兩個模型，在 8 個由淺入深的投資問答中之表現。  
目標是評估兩者在 **結構、專業度、可用性、合規性** 等面向的差異。

- **Baseline**：提供大量數字與情境，但偶有口徑不一、帶入「最可能」等預測語氣，合規風險較高。  
- **Persona**：假設清楚、框架一致、語氣合規且數據自洽，偏教育性工具化輸出。

> **免責聲明**：本報告僅供金融教育與模擬示範之用，不構成投資建議。

## 🔑 Key Differences

| 評估維度 | Baseline | Persona |
| --- | --- | --- |
| **資料正確性** | 混入「當下」收益/主張與情境數字，並以「最可能」語氣敘述。 | 嚴格以假設與歷史平均作示範，避免即時數據主張，數字自洽。 |
| **計算嚴謹度** | 基本複利正確；股息「緩衝」等段落未完整推導或列式。 | 公式/程式碼清楚，輸入條件與步驟可重現。 |
| **結構與清晰度** | 敘事長、資訊量大但較散；雖有情境卻不夠模組化。 | 穩定框架：**Assumptions → Framework/Scenarios → Calculation → Risks → TBC**。 |
| **合規性** | 多次出現建議/預測語氣（如「更適合」「最可能」）。 | 全程教育性與免責聲明，避免個別建議或預測。 |
| **需求貼合度** | 直接回答需求、給數字。 | 在回答之外，提供決策框架/檢核表，契合「成長+容忍波動」設定。 |
| **一致性** | 論述與語氣在不同題次間波動。 | 語氣、邏輯、數據處理前後一致。 |
| **可操作性** | 有情境與結論，但較少「如何自行檢核」工具。 | 明確「下一步」與檢核重點（prospectus、30-Day SEC Yield、年度回顧）。 |

## 🧮 Methodology
  
評分公式：每個構面先給 **1–5 分**，再依 **權重 × (分數/5)** 換算加權分，合計 **0–100 分**。  
扣分準則： 
- 出現「**最可能**」「**更適合**」等帶建議/預測語氣 → **合規性**重扣。  
- 以「口述估計」取代明確公式/推導 → **計算嚴謹度**扣分。  
- 混用即時數據或含糊來源 → **數據正確性**扣分。  
- 結構鬆散、缺少 Assumptions / Risks / TBC → **結構與清晰度**扣分。  
- 前後口徑不一致 → **一致性**扣分。  
- 僅給數字、缺少可落地的檢核/決策工具 → **可操作性**扣分。 

**權重**：資料正確性 20、計算嚴謹度 15、結構清晰度 15、合規性 15、需求貼合度 15、一致性 10、可操作性 10。

## 🏁 Verdict（結論）
經過計算，**Persona 明顯勝出**：  
- 以可重現的數學框架與清晰結構回應提問，並在 **合規性、結構與工具化可用性** 上大幅領先。  
- **Baseline** 雖快速直接、數字豐富，但多處採用判斷式語氣（如「最可能」「更適合」），且部分推導未完整，導致 **合規與嚴謹度** 扣分。

## ✅ Reviewer Notes & Next Steps（改進建議）
**對 Persona：**  
- 加入 **序列風險（Sequence of Returns）** 與 **蒙地卡羅** 範例；用區間/分位數展示不確定性。  
- 於股息、費率、稅負、通膨等假設列「**敏感度分析**」小表（±1–2% 變動）。  
- 提供「**自檢清單**」：Prospectus 連結、30-Day SEC Yield、Top 10 Holdings、Sector Weights、年度檢視節點。

**對 Baseline：**  
- 移除「**最可能/更適合**」等主觀語氣；改以「**在 X 條件下**，結果可能落在 Y–Z 區間」。  
- 凡提及「股息緩衝/現金流」→ **務必列公式與計算步驟**，避免口述估計。  
- 用 Persona 的結構化模板包裝輸出（Assumptions → Scenarios/Method → Calculation → Risks → TBC）。

## 📎 附錄：評分口徑（節選）
- **合規性**：避免個別化建議與預測語；強化「教育用途」與「非財務建議」之聲明。  
- **計算嚴謹度**：能列式就不口述；能程式就不手算；能給區間就不給單點。  
- **可操作性**：把「數字」轉成「工具」：決策表、檢核表、檢視節點、資料來源查核步驟。
